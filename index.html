Interactive Trust Simulator

Why Do People Cooperate or Betray?

This page is a simple lab where digital agents repeatedly choose to cooperate or defect. You can change settings, run experiments, and immediately see how trust grows, breaks, or recovers.

IPD = Iterated Prisoner's Dilemma MARL = Multi-Agent Reinforcement Learning Runs fully in your browser

Start Here (Non-Technical)

The Game

Two players choose each round:

Cooperate (help both) or Defect (try to exploit).

One round pushes toward defection. Repeated rounds can make cooperation rational.

What You Control

You pick number of rounds, noise (mistakes), memory length, and learning behavior.

Then click Start Simulation and watch trust metrics update live.

What You Learn

When does cooperation emerge? When does it collapse into mutual punishment?

Use the comparison panel to test "before vs after" settings side-by-side.

Tip: First run with defaults, then only change one setting at a time for clear comparisons.

Agent Guide (Who Is Who)

Q-learning Agent

Learns from reward over time. It starts uncertain and adapts strategy based on experience.

Always Cooperate (AllC)

Always chooses cooperation. Kind, but easy to exploit.

Always Defect (AllD)

Always betrays. Strong in one-shot play, often poor long-term socially.

Tit For Tat (TFT)

Starts nice, then copies your last move. Cooperative but retaliatory.

Grim Trigger

Cooperates until betrayed once, then defects forever.

Copykitten

Defects only after two consecutive betrayals. More forgiving under noise.

Random

Chooses cooperate/defect randomly. Unpredictable baseline.

What Happens in the Background

  1. Agents repeatedly play Cooperate/Defect using the same payoff rules.
  2. Rewards are computed from the Prisoner's Dilemma matrix after each round.
  3. Learning agents update their strategy values (Q-learning updates).
  4. Your browser draws the charts and stores run history locally for comparison.

No server is required: this simulation runs entirely on your device.
